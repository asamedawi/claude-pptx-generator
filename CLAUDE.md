# PowerPoint Generator

このリポジトリでは、`templates/` 内の PowerPoint テンプレートを使って資料を作成する。

## ルール

- 使用するテンプレートは `templates/template.pptx`
- 出力先は `output/output.pptx`
- ユーザーからは `「○○についての資料を作成してほしい。」` のような短い依頼だけが来る想定とする
- テーマはユーザーの依頼文から読み取る
- 資料作成を始める前に、AskUserQuestion 機能を使って必要事項をユーザーに質問する
- AskUserQuestion の質問には選択肢を設定する
- 目的、対象者、所要時間は選択式で確認する
- `references.md` が存在する場合は、資料内容の作成時の参考情報として使う
- ユーザーが依頼文の中で追加の参考文献・リンクを明示した場合は、それも参考情報として使う
- 質問結果と `references.md` の内容をもとに骨子を作成し、プランモードでユーザー確認後にスライド内容を作成する
- プランモードで合意した内容は、生成前に Markdown 形式で整理する
- 資料は原則として `導入 → 本題 → まとめ` の3部構成にする
- 1枚目では、少なくともテーマと目的が伝わるタイトルにする
- 2枚目では、資料全体の流れが分かるアジェンダを示す
 - ただし、短い資料では省略してよい
- 本題は 3 つ以内の論点に整理する
- 1スライドでは 1 つのメッセージだけを伝える
- スライドタイトルは単なる項目名ではなく、そのスライドで伝えたい結論を書く
- まとめスライドでは、要点を 3 つ以内で整理する
- 最後に、聞き手にしてほしい次の行動や確認事項を明記する
- 提案資料では `課題 → 提案 → 効果 → 次のアクション` を基本構成とする
- 情報共有資料では `結論 → 根拠 → 詳細 → まとめ` を基本構成とする
- `generate_pptx.py` 実行時に `--no-open` フラグは付けない（生成完了後に必ずアプリで開く）

## 依頼テンプレート

`templates/template.pptx` を使って PowerPoint 資料を作成する。
出力先は `output/output.pptx`。
ユーザーから依頼を受けたら、いきなり資料を作り始めず、まず依頼文からテーマを読み取る。
そのうえで、AskUserQuestion 機能で以下を確認する。

- 目的
  - 情報共有
  - 提案
- 対象者
  - 顧客
  - 社内
- 所要時間
  - 5 分
  - 10 分
  - 15 分
  - 30 分

質問への回答と `references.md` の内容をもとに、まず以下の骨子を作成する。

- テーマ：○○
- 目的：○○
- 対象者：○○
- 所要時間：○○ 分
- 参考文献：`references.md` および依頼文で指定された情報（任意）

骨子はプランモードでユーザーに確認してもらう。
骨子が固まってから、タイトルと本文を整理し、見やすいスライド構成にする。
`references.md` やユーザー指定の参考文献がある場合は、その内容をもとにスライドのタイトルや本文を具体化する。
必要に応じて、参考文献として末尾スライドや各スライド内に出典を反映してよい。
生成直前に、確定したスライド内容を以下の形式の Markdown として整理する。

- 1 行目は `# 資料タイトル`
- 2 ページ目以降は `## スライドタイトル`
- 各スライドの本文は `- 箇条書き`
- 参考文献を追加する場合は、末尾に `## 参考文献` を作り、`- タイトル: URL` 形式で整理する
