# PowerPoint Generator

このリポジトリでは、`templates/` 内の PowerPoint テンプレートを使って資料を作成する。

## ルール

- 使用するテンプレートは `templates/template.pptx`
- 出力先は `output/output.pptx`
- ユーザーからは `「○○についての資料を作成してほしい。」` のような短い依頼だけが来る想定とする
- テーマはユーザーの依頼文から読み取る
- 資料作成を始める前に、AskUserQuestion 機能を使って必要事項をユーザーに質問する
- AskUserQuestion の質問には選択肢を設定する
- 目的、対象者、所要時間は選択式で確認する
- 参考文献・リンクは任意項目として確認し、`なし` を選ばない場合は複数追加できるようにする
- `URL を指定する` や `その他` が選ばれた場合は、1件受け取っただけで終了せず、追加有無を確認しながら複数件収集する
- 参考文献・リンクの収集が完了したとユーザーが明示するまでは、骨子作成やプランモードに進まない
- 参考文献・リンクが提供された場合は、資料内容の作成時の参考情報として使う
- 質問結果をもとに骨子を作成し、プランモードでユーザー確認後にスライド内容を作成する
- プランモードで合意した内容は、生成前に Markdown 形式で整理する
- `generate_pptx.py` 実行時に `--no-open` フラグは付けない（生成完了後に必ずアプリで開く）

## 依頼テンプレート

`templates/template.pptx` を使って PowerPoint 資料を作成する。
出力先は `output/output.pptx`。
ユーザーから依頼を受けたら、いきなり資料を作り始めず、まず依頼文からテーマを読み取る。
そのうえで、AskUserQuestion 機能で以下を確認する。

- 目的
  - 情報共有
  - 提案
  - その他
- 対象者
  - 顧客
  - 社内
  - その他
- 所要時間
  - 5分
  - 10分
  - 15分
  - 30分
  - その他
- 参考文献・リンク（任意、`なし` 以外は複数選択可）
  - なし                                
  - URL を指定する
  - その他
- `なし` を選ばない場合は、URL や参照元を複数受け付けてよい
- `URL を指定する` または `その他` を選んだ場合は、1件入力されるたびに「さらに追加するか」を確認し、ユーザーが完了を示すまで繰り返す
- 参考文献・リンクの追加確認が終わる前に、骨子作成やプランモードへ進めてはならない

質問への回答をもとに、まず以下の骨子を作成する。

- テーマ：○○
- 目的：○○
- 対象者：○○
- 所要時間：○○分
- 参考文献・リンク：○○（任意）

骨子はプランモードでユーザーに確認してもらう。
骨子が固まってから、タイトルと本文を整理し、見やすいスライド構成にする。
参考文献・リンクが提供された場合は、その内容をもとにスライドのタイトルや本文を具体化する。
必要に応じて、参考文献として末尾スライドや各スライド内に出典を反映してよい。
生成直前に、確定したスライド内容を以下の形式の Markdown として整理する。

- 1行目は `# 資料タイトル`
- 2ページ目以降は `## スライドタイトル`
- 各スライドの本文は `- 箇条書き`
- 参考文献を追加する場合は、末尾に `## 参考文献` を作り、`- タイトル: URL` 形式で整理する
